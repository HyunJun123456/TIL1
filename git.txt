git
: 깃이 제대로 깔렸는지 알려줌

깃 환경 설정하기
git config --global user.name "NAME"
git config --global user.email "Email@a.com"
: 깃에게 사용자 정보 알려주기

현재 디렉터리 살펴보기
pwd
: print working directory
물결 표시는 홈 디렉터리에 있는 것을 의미
현재 위치의 경로를 알려줌

ls
: 현재 디렉터리에 어떤 파일이나 디렉터리가 있는지 확인
이름 뒤에 슬래시가 붙어 있는 것이 디렉터리임
list의 줄인말

ls 명령 옵션 모음
-a : 숨긴 파일과 디렉터리도 함께 표시
-l : 파일이나 디렉터리의 상세 정보를 함께 표시
-r : 파일의 정렬 순서를 거꾸로 표시
-t : 파일 작성 시간 순으로 (내림차순) 표시

터미널 창에서 디렉터리 이동하기
cd ..
: 상위 디렉터리로 이동
cd Users
: 유저 디렉터리로 이동
cd ~
: 홈 디렉터리로 이동
cd는 change directory의 줄인말

리눅스에서 디렉터리를 나타내는 기호
~ : 현재 접속 중인 사용자의 홈 디렉터리를 가리킴
./ : 현재 사용자가 작업 중인 디렉터리
../ : 현재 디렉터리의 상위 디렉터리

터미널 창에서 디렉터리 만들기 및 삭제하기
mkdir test
: test라는 디렉터리 생성
make directory의 줄인말

rm -r test
: test라는 디렉터리 삭제
rm은 remove의 줄인말

빔에서 텍스트 문서 만들기
cd Documents
mkdir test
cd test
vim test.txt
새로 만든 파일에는 제대로 작동을 안함
I 또는 A를 눌러서 입력모드로 변경해야함
I는 insert, A는 add의 약자
esc를 누르면 ex모드로 다시 돌아감
콜론(:)을 입력하면 명령코드를 작성할 수 있음
:wq -> w는 저장, q는 종료를 의미

빔 ex모드 명령 모음
:w 또는 :write - 편집 중이던 문서를 저장
:q 또는 :quit - 편집기 종료
:wq(파일) - 편집 중이던 문서를 저장하고 종료(파일 이름을 입력하면 그 이름으로 저장)
:q! - 문서를 저장하지 않고 편집기를 종료

텍스트 문서 내용 확인하기
cat test.txt
: 텍스트 파일의 내용을 보여줌

cat명령
cat 파일 : 파일의 내용을 화면에 표시
cat 파일1, 파일2 ''' 파일n > 새파일 : 파일 n개를 차례로 연결해서 새로운 파일을 만듦
cat 파일1>>파일2 : 파일1의 내용을 파일2 끝에 연결

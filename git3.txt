원격 저장소에 연결하기
git remote add origin 복사한 주소 붙여넣기
: 원격 저장소(remote)에 origin을 추가(add)하겠다고 깃에게 알려주는 거임
origin은 깃허브 저장소 주소를 나타내는데 줄여서 저렇게 씀
깃에서 기본 브랜치를 master라고 하는 것처럼 기본 원격 저장소에는 origin이라는
이름을 사용함

git remote -v
: 원격 저장소에 제대로 연결됐는지 확인해 볼 수 있음

원격 저장소에 파일 올리기
git push
: 지역 저장소의 커밋을 원격 저장소로 보내는 푸시

git push -u origin master
: 지역 저장소의 브랜치를 origin, 즉 원격 저장소의 master 브랜치로
푸시하라는 명령어. 여기에서 -u 옵션은 지역 저장소의 브랜치를 
원격 저장소의 master 브랜치에 연결하기 위한 것으로 처음 한 번만 사용하면 됨

한번 커밋한 이력이 있다면 추가할때 -am 옵션을 사용할 수 있음

원격 저장소에서 파일 내려받기 
git pull
: 원격 저장소에 있는 소스 파일을 다른 사용자가 수정했거나 깃허브 사이트에서
직접 커밋하면 지역 저장소와 차이가 생김
원격 저장소와 지역 저장소의 상태를 같게 만들기 위해 원격 저장소의
소스를 지역 저장소로 가져옴

소스를 수정하고 커밋을 하지 않은 상태에서 원격저장소의 내용이 변경 되었을 때
소스를 add하고 commit까지 하는건 가능하고 push가 되지 않는데 이때 pull을 해줘야함
소스가 다른건 깃이 알아서 합쳐주고 같은 곳에 작성되면 알려주게 됨 
그러니 소스가 없어질 걱정이 없음

git pull origin master
: 원격 저장소의 내용을 master 브랜치로 가져온다는 의미인듯
